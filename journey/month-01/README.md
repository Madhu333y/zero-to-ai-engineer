# Month 01 — Python Runtime & Foundations

## 🎯 Focus

Understanding how Python actually runs, fails, and behaves in a real environment.

---

## 🧠 What I Learned

* How Python executes code step-by-step
* Terminal workflow and running scripts
* Virtual environments (`venv`)
* Dependency installation and management
* Reading and understanding error messages
* Basics of debugging and logging

---

## 🧑‍💻 What I Built

* Small Python scripts to test execution flow
* Input → process → output programs
* Intentionally broke code to understand failures

---

## 💥 What Broke

* Missing dependencies
* Incorrect imports
* Runtime errors due to wrong inputs
* Confusion around how Python handles execution

---

## 🔧 Key Insights

* Understanding how code runs is more important than writing more code
* Debugging is a core skill, not a side skill
* Errors are signals — not problems
* Environment setup matters more than expected

---

## 🚀 How This Helped

This month built the foundation for backend development.

Because of this:

* FastAPI concepts are easier to understand
* Debugging API issues is more manageable
* I can read and trace errors more confidently

---

## ⚡ Next Step

Move to backend systems with FastAPI:

* API structure
* Request/response flow
* Validation and error handling
